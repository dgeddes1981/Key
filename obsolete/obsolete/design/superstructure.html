superstructure overview

The system superstructure is that broad framework that ties Atoms together.  By way of contrast, the system substructure is the makeup of the Atoms themselves.

There are several basic classes

  • Factory
      • A singleton that controls the construction of Atoms
  • Registry
      • A singleton that assosciates Atoms with their indexes.  It also manages loading and saving Atoms to disk.
  • Reference(s)
      • A reference is a persistent method for pointing to an Atom.
  • Atom(s)
      • The building block of the core makeup of the system.
There are several basic operations that this superstructure takes care of:
  1. Creating a new standalone Atom
      2.  
  2. Creating a new part atom
  3. Loading a distinct atom from disk
  4. Swapping a distinct atom to disk
  5. Registering an atom part
  6. De-registering an atom part
  7. Registering a temporary atom
  8. De-registering a temporary atom
  9. Loading the core system from disk
  10. Writing the core system to disk
  11. Destroying an atom
